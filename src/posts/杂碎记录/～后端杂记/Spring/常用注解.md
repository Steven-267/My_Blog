# Springboot 的常用注解（持续更新。。。）
``` java
1. @RestController：@Controller 和 @ResponseBody  的组合注解。每个方法的返回值都会以 JSON 或 XML 的形式直接写入 HTTP 响应体中，相当于在每个方法上都添加了 @ResponseBody 注解。
2. @Controller：适用于传统的 MVC 架构，它负责处理请求并返回相应的视图。（@RestController下的方法默认返回的是数据格式，@Controller注解标注的类下面的方法默认返回的就是以视图为格式）
3. @RequestMapping：将请求和处理请求的控制器方法关联起来，建立映射关系。（说白了就是定义一个请求到哪的路径）
4. @reponseBody：用于将controller的返回结果经过转化转为json格式
5. @pathvariable：获取get请求中的参数
6. @GetMapping：用于细化接口的请求类型
7. @PostMapping：用于细化接口的请求类型
8. @SpringbootApplication：用于标记主配置类，其中包含了@SpringBootConfiguration、@EnableAutoConfiguration、@ComponentScan三个注解
9. @SpringBootConfiguration：表示这是一个 Spring 配置类，它等价于 @Configuration 注解，用于声明类作为 Spring 的配置类。
10. @EnableAutoConfiguration：启用 Spring Boot 的自动配置功能，Spring Boot 会根据添加到项目中的依赖库来自动配置应用程序，它可以帮助减少显式的配置代码，因为 Spring Boot 会自动检测和配置合适的 Bean。
11. @ComponentScan：用于指定要扫描的包，以便 Spring 可以在这些包中查找带有 @Component、@Service、@Repository 和 @Controller 等注解的类。默认情况下，@ComponentScan 会在包含 @SpringBootApplication 注解的类所在的包及其子包中进行扫描。
12. @Autoware: 用于实现依赖注入,允许 Spring 自动地将匹配的 Bean 注入到类的字段、构造函数或方法中。
13. @Resource： Java 的一个注解，用于实现依赖注入，和@Autoware类似，但与之不同的是他默认通过name匹配来注入Bean，如果找不到匹配的name则会退回到通过类型匹配。
。。。
```
